# CyberLeninka Parser

Парсер научных статей с [cyberleninka.ru](https://cyberleninka.ru).  
Собирает метаданные и полный текст статей по заданным темам, сохраняет в CSV/JSON.  
Файлы пишутся в папку **`data/`** в корне репозитория (общая с новостными сборщиками).

## Быстрый старт

```bash
cd cyberleninka

python3 -m venv .venv

source .venv/bin/activate

pip install -r requirements.txt

python3 main.py --topics "Право,Математика" --pages 10
```

### Проанализировать и очистить результат

```bash
# Базовый анализ + очистка по умолчанию
# (файлы лежат в data/ в корне репозитория; из папки cyberleninka — ../data/...)
python3 analyze.py --input ../data/cyberleninka_articles_20260307_124155.csv

# Максимальная очистка: все артефакты включая сноски, ключевые слова и ссылки
python3 analyze.py --input data/cyberleninka_articles_20260307_124155.csv \
    --inline-refs --keywords-line --footnote-nums

# Только статистика, без очистки
python3 analyze.py --input data/cyberleninka_articles_20260307_124155.csv --no-clean

# Только очистить (без повторного парсинга)
python3 main.py --clean-only --input data/cyberleninka_articles_20260307_124155.csv

# Повторная очистка с максимальными параметрами
python3 main.py --clean-only --input data/cyberleninka_articles_20260307_124155.csv \
    --inline-refs --keywords-line --footnote-nums
```

## Параметры main.py

### Парсинг

| Параметр | Тип | По умолч. | Описание |
|---|---|---|---|
| `--pages N` | int | 60 | Страниц API на тему. Итого = pages × 10 статей |
| `--topics STR` | str | 5 предустановленных | Темы через запятую |
| `--filter STR` | str | none | Фильтр: `none` / `rsci` / `vak` / `scopus` |
| `--delay SEC` | float | 1.2 | Задержка между запросами в секундах |
| `--pdf` | flag | выкл. | Скачивать PDF при отсутствии HTML-текста |
| `--no-json` | flag | выкл. | Не сохранять JSON |
| `--clean` | flag | выкл. | Сразу применить очистку текста |
| `--clean-only` | flag | выкл. | Только очистить готовый CSV (парсер не запускать) |
| `--input FILE` | str | — | Входной CSV для `--clean-only` |

### Опции очистки (для `--clean` и `--clean-only`)

| Параметр | По умолч. | Описание |
|---|---|---|
| `--no-join-hyphens` | выкл. | Не склеивать OCR-переносы «слово-\nпродолжение» |
| `--inline-refs` | выкл. | Удалять ссылки `(1)`, `[2]` внутри текста |
| `--keywords-line` | выкл. | Удалять строку «Ключевые слова: …» из тела текста |
| `--footnote-nums` | выкл. | Удалять нумерацию сносок `1.`, `[1]` в начале строк |

## Очистка текстов

`analyze.py` и `main.py --clean` автоматически удаляют из текстов:

| Артефакт | Пример | Флаг |
|---|---|---|
| BOM-символы | `﻿` в начале текста | всегда |
| Email-адреса | `myagkova.post@ gmail.com` | всегда |
| Строки «E-mail:» | `E-mail: author@mail.ru` | всегда |
| URL-ссылки | `https://www.site.ru/...` | всегда |
| URL с OCR-пробелом | `http:// www.site.ru/...` | всегда |
| «Режим доступа:» | `— Режим доступа: http://...` | всегда |
| «Дата обращения:» | `(Дата обращения: 28.06.2016.)` | всегда |
| УДК/ББК/DOI строки | `УДК 140.8  ББК 6/8` | всегда |
| DOI в теле | `10.24411/1994-2796-2018-11118` | всегда |
| Список литературы | `Список литературы\n1. Автор...` | всегда |
| Фрагменты URL-путей | `councils/chrstuni/doc_en.html` | всегда |
| Блоки «Сведения об авторах» | Блоки аффилиаций в конце | всегда |
| Мягкие переносы | `\xad` (из PDF) | всегда |
| Управляющие символы | `\x0c`, `\x08` и др. | всегда |
| Неразрывные пробелы | `\u00a0` → обычный пробел | всегда |
| OCR-переносы | `иудей-\nско` → `иудейско` | `по умолч. вкл.` |
| Ссылки `(1)`, `[2]` в тексте | `...влияния (1) на...` | `--inline-refs` |
| Строка «Ключевые слова:» | `Ключевые слова: теология` | `--keywords-line` |
| Нумерация сносок | `1.\t`, `[1]` в начале строки | `--footnote-nums` |

> **Осторожно: `--keywords-line`**  
> Часть OCR-текстов хранится как **одна длинная строка** без переносов `\n`. В таком формате паттерн «Ключевые слова: …» может совпасть уже в середине текста и удалить всё до конца строки. Флаг ограничен 2000 символами, поэтому для нормальных текстов безопасен. Если вы видите `text_length_clean` подозрительно маленьким — скорее всего текст был однострочным и «Ключевые слова:» стояло сразу за заголовком. В этом случае лучше не использовать `--keywords-line`.

## Как получить полный текст статей

CyberLeninka не предоставляет API для текста. Два способа:

### 1. HTML OCR (быстро, ~1–2 с/статья)

Текст берётся с HTML-страницы статьи из блоков `div.ocr`, `div[itemprop="articleBody"]` и др.  
Покрытие: ~60–80% статей. Иногда текст обрезан или содержит OCR-артефакты.

```bash
python3 main.py --pages 10
```

### 2. PDF (медленно, ~5–15 с/статья)

Если на HTML-странице текст не найден, скачивает PDF и извлекает текст через `pdfminer.six`.  
Покрытие выше, но время выполнения значительно увеличивается.

```bash
python3 main.py --pages 10 --pdf
```

## Настройка тем и параметров по умолчанию

Все настройки находятся в `src/config.py`:

```python
TOPICS = [
    "Философия, этика, религиоведение",
    "История и археология",
    "Математика",
    "Право",
    "Политологические науки",
]

MAX_PAGES      = 60     # 60 × 10 = 600 статей/тему
FILTER         = None   # None | 22 (РИНЦ) | 8 (ВАК) | 2 (Scopus)
DELAY          = 1.2    # задержка между запросами, сек.
FETCH_PDF_TEXT = False  # загружать ли PDF при отсутствии HTML-текста
```

Изменять `config.py` или переопределять через аргументы — оба способа работают.
